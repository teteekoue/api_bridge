ÉTAT DU PROJET : ARCHITECTURE ASYNCHRONE (POLLING) - STABLE
- Architecture : Système de Polling (/ask pour créer, /result pour récupérer) éliminant les timeouts HTTP "Aborted".
- Stratégie Automation : "Click & Verify" mécanique. Swipe 3s -> Clic aveugle -> Vérification du changement de presse-papier.
- Détection de fin : Validée par la réussite physique de l'action de copie, indépendante de l'analyse du code source UI.
- Timeouts : Étendus à 3 heures (10800s) pour supporter les générations d'IA très longues.
- Interface Web : Dashboard mis à jour avec gestion automatique du cycle de vie du Job (Polling JS).
- Robustesse : Suppression de la latence liée à l'empreinte globale de l'écran.
